Serverkonfiguration

Alles was du wissen musst, um deinen Server zu konfigurieren!

Server-Konfigurationsdatei​

tip

nanos world Config Dateien verwenden TOML (Toms offensichtlich, Minimal Language), bitte lesen Sie https://github.com/toml-lang/toml für weitere Informationen und Syntax.

Die Server-Konfigurationsdatei Config.toml wird automatisch erzeugt, wenn der Server zum ersten Mal gestartet wird. Diese Datei wird immer mit dem korrekten Muster überschrieben, nachdem der Server geladen wurde.

Server.toml
# server configurations
[server]
    # server name
    name =                  "nanos world Server"
    # server description (max 250 characters)
    description =           ""
    # server logo URL (recommended size: 150x75)
    logo =                  "https://i.imgur.com/U1rZp5v.png"
    # max players
    max_players =           60
    # leave it blank for no password
    password =              ""
    # server IP. we recommend leaving it 0.0.0.0 for default
    ip =                    "0.0.0.0"
    # server port (TCP and UDP forwarding needed)
    port =                  7777
    # query port (UDP forwarding needed)
    query_port =        7778
    # announce server in the master server list
    announce =              true
    # server tick rate in milliseconds (dangerous! -Server wird auf jeden [tick_rate] ms überprüft, was sowohl die Server- als auch die Clientleistung beeinflusst. 33 ms bedeutet 30 Ticks pro Sekunde und ist der Standardwert und der empfohlene Wert)
    tick_rate = 33
    # Log-Level. (1) normal - (2) debug - (3) verbose
    log_level = 2

# nanos world configurations
[world]
    # game-mode package to load (set the main game-mode package to load - you can load only one 'game-mode' package type at once)
    game_mode = ""
    # loading-screen package to load (the loading screen will be shown when players join your server)
    loading_screen = ""
    # packages script list (leave it blank to load all packages, Dies ist normalerweise nützlich, wenn Sie tonnenweise heruntergeladene Pakete heruntergeladen haben und nur ein oder einige laden möchten)
    packages = [

    ]
    # Liste der Asset-Packs (dies wird zusätzlich die Asset-Packs die Sie hier definieren)
    Assets = [

    ]
    # Standard Startup Map
    map = "nanos-world::BlankMap"
    # gebannte IPs
    banned_ips = [

]

Karte & Level​

Die Karte (oder Ebene) ist in der Konfigurationsdatei des Servers definiert dieses Level wird geladen, wenn der Spieler dem Server beitritt, und der Pfad soll ein eingebautes Asset sein oder ein Asset sein, das sich im Ordner Assets/ befindet.

nanos world zählt auf (für jetzt) 2 eingebaute Karten: nanos-world::BlankMap und nanos-world::TestingMap , die auf Ihrem Server verwendet werden kann, ohne ein Asset Pack herunterladen zu müssen.

    Package.Log("Console command received: " .. my_input)

Allgemeine Konsolen-Nachrichten/Warnungen/Erros und seine Bedeutung​

Server zu stark/extrem hoch! Überprüfen Sie die Leistung des Servers! Server steckte für Xms fest.​

Das bedeutet, dass der Server für X Millisekunden (laggy) stecken geblieben ist. Die Warnung (yellow) ist kein Grund zur Sorge. aber zu viele Fehler (rot) könnten bedeuten, dass Ihre Serverinfrastruktur nicht so gut ist oder Ihr Skriptcode nicht so optimiert ist. Der Server versucht 1 läuft mit 33 Ticks pro Sekunden (oder dem Betrag der auf Config.toml). Der Server läuft in einer Endlosschleife, die in einer Frequenz von 1/33 Millisekunden liegt. Innerhalb dieser Schleife werden alle Serveroperationen ausgeführt: Netzwerkpakete empfangen/senden, Funktionen (vom Netzwerk empfangen) ausführen das Auslösen von Skript-Ereignissen, das Ausführen aller Ticks-Ereignisse, die Berechnung von Trigger-Überlappungen usw. Wenn also eine dieser Operationen mehr als 1/33 Millisekunden benötigt, wird diese Warnung angezeigt.

Ich sagte tries1 weil bei Fenstern, die nicht so genau sind als bei Linux aufgrund interner c++ Implementierungen.
